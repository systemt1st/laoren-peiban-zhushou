# 老人陪伴助手 PRD

## 1. 文档信息
- 产品名称：老人陪伴助手
- 文档版本：v1.0（MVP）
- 更新时间：2026-03-21
- 目标终端：手机 Web（H5，浏览器访问）
- 交互主轴：语音为主，文字为辅

## 2. 产品定位
### 2.1 核心定位
面向 60+ 老年用户，提供“开口即用”的语音陪伴与生活助理能力；在识别到风险表达时，快速进入应急引导路径，并保留关键记录供家属与后台追溯。

### 2.2 产品愿景
降低独居老人孤独感与生活风险门槛，让语音成为最自然的入口，实现“像孙子孙女一样陪聊，关键时刻能救命”。

### 2.3 产品目标（MVP）
- 陪伴与问答：可稳定进行多轮语音对话，支持常见生活类问答。
- 应急引导：识别高风险表达，触发“紧急模式”，输出清晰、可执行步骤。
- 基础画像：支持最小化用户偏好与紧急联系人信息管理。
- 可追溯：形成对话、提醒、应急事件记录，支持后台/家属查询。
- 合规安全：不做医疗诊断，风险场景优先引导求助。

## 3. 目标用户与核心场景
### 3.1 目标用户
- 主用户：60 岁以上老年人（独居/半独居）。
- 次用户：家属（查看关键记录、联系人维护）。
- 运营用户：后台客服/运营（事件复核、风险回访）。

### 3.2 核心场景
- 日常陪伴：用户随时说“聊聊天”，系统持续回应与情绪陪伴。
- 生活提醒：吃药、喝水、复诊提醒，降低遗忘风险。
- 信息问答：天气、时间、节日、生活常识等低门槛查询。
- 突发不适：出现“胸痛、呼吸困难、摔倒”等表达时启动应急引导。
- 家属追溯：查看近期应急事件、提醒完成情况、关键对话摘要。

## 4. 需求范围与功能清单
### 4.1 MVP 功能清单（必须）
1. 语音对话
- 一键开始/停止听写。
- 语音识别转文字展示。
- 文字与语音双通道回复（TTS 播报）。
- 连续多轮上下文（会话内）。

2. 风险识别与紧急模式
- 关键词 + 规则初筛（如胸痛、晕厥、呼吸困难、摔倒、流血不止）。
- 触发紧急模式后高优先展示：
  - 立即拨打急救电话建议（120）。
  - 联系紧急联系人入口。
  - 当前状态确认（是否可自行行动、是否有人在旁）。
- 记录应急事件（时间、触发词、引导步骤、用户操作）。

3. 提醒管理
- 提醒创建：标题、时间、频率、备注（可选）。
- 提醒列表：待执行/已完成。
- 简单确认：完成、稍后提醒。

4. 基础画像与联系人
- 老人资料：称呼、年龄段、常住城市、偏好话题。
- 紧急联系人：姓名、关系、手机号、优先级。

5. 记录与追溯
- 对话记录列表（时间、主题、风险标记）。
- 应急记录列表（级别、动作、结果）。
- 面向家属/后台的查询接口约束（仅授权访问）。

### 4.2 P1 功能（下一阶段）
- 语音唤醒词与免手动点击模式（浏览器能力允许时）。
- 提醒到点自动语音播报。
- 家属通知（短信/IM 网关）。
- 应急脚本按症状分层（非诊断，仅流程化引导）。

### 4.3 P2 功能（中期）
- 个性化陪伴（记忆用户偏好与重要日期）。
- 多语言/方言适配。
- 与可穿戴设备联动（心率异常事件输入）。

## 5. 信息架构（IA）
### 5.1 一级导航
- 首页（语音入口）
- 对话（历史与实时）
- 提醒
- 应急
- 我的（画像、联系人、设置）

### 5.2 关键对象
- UserProfile（用户画像）
- EmergencyContact（紧急联系人）
- ConversationSession（会话）
- ConversationMessage（消息）
- Reminder（提醒）
- EmergencyEvent（应急事件）

### 5.3 页面间关系
首页是高频入口，向“对话”和“应急”分流；“提醒”独立承载任务型行为；“我的”负责低频配置；“记录”由“对话/应急”分域承载并统一检索。

## 6. 关键用户流程
### 6.1 首次使用流程（老人端）
1. 进入 H5 后展示大按钮“按住说话/点击开始”。
2. 完成最小资料设置：称呼 + 至少 1 个紧急联系人。
3. 进行首次语音测试，确认识别与播报可用。
4. 引导完成第一个提醒创建（如喝水提醒）。

### 6.2 日常陪聊流程
1. 用户点击语音按钮发起输入。
2. 系统识别语音并回显文字，进入对话引擎。
3. 输出文字 + 语音回复。
4. 对话落库并打标签（普通/情绪/风险）。

### 6.3 应急流程
1. 用户表达不适或系统命中风险规则。
2. 立即切换“紧急模式”UI（高对比、低干扰）。
3. 分步引导：先求助再描述，优先电话动作。
4. 用户可一键拨打 120 或紧急联系人。
5. 事件全程记录并可供家属/后台追溯。

### 6.4 家属查询流程（Web 管理侧最小版）
1. 登录后选择绑定老人。
2. 查看近 7 天应急事件与提醒完成率。
3. 点进事件详情查看触发词与处理动作时间线。

## 7. 交互原则与可用性规范
### 7.1 交互原则
- 低门槛：一屏一个主动作，减少学习成本。
- 可确认：关键动作必须二次确认（如呼叫联系人）。
- 可回退：误触可撤回，避免焦虑。
- 强反馈：录音中、识别中、已发送状态明确展示。

### 7.2 视觉与可读性
- 默认字号 >= 18px，按钮高度 >= 48px。
- 高对比配色，避免浅灰文字。
- 关键按钮固定在拇指可达区（底部主操作）。

### 7.3 语音交互细则
- 允许语音 + 文字混输。
- 语音失败时自动回退到文字输入建议。
- TTS 支持“慢速播报”和“重复一次”。

## 8. 核心页面设计草案
### 8.1 首页（语音入口）
- 模块：问候语、超大语音按钮、快捷入口（提醒/应急/联系人）。
- 状态：待机、录音中、识别中、播报中。

### 8.2 对话页
- 模块：消息流、语音输入按钮、重播按钮、文字输入兜底框。
- 重点：最近一条回复突出显示，避免信息拥挤。

### 8.3 提醒页
- 模块：今日提醒、周期提醒、快速新增。
- 行为：完成、稍后 10 分钟、删除（需确认）。

### 8.4 应急页
- 模块：高风险说明、一步一步引导卡片、快速呼叫按钮。
- 重点：屏蔽无关信息，保留最短求助路径。

### 8.5 我的页
- 模块：基本资料、紧急联系人、语音设置、隐私授权说明。

## 9. 风险与合规
### 9.1 合规边界
- 明确声明“非医疗诊断产品”。
- 医疗相关问题默认建议联系医生或急救渠道。
- 不输出处方、剂量调整等专业诊疗结论。

### 9.2 数据与隐私
- 最小必要采集原则（仅采集业务必需字段）。
- 对话与事件记录需标记敏感级别。
- 联系人手机号等敏感字段加密存储（服务端）。
- 明确告知数据用途、保留周期与删除方式。

### 9.3 风险控制
- 误报：可人工取消紧急模式并记录原因。
- 漏报：对高危词保持高召回优先，后续持续调优词典。
- 网络失败：前端展示“离线求助建议”与本地紧急号码。

## 10. 埋点与指标体系
### 10.1 核心北极星指标
- 日活跃老人用户数（DAU-elder）。
- 每用户日均有效语音轮次。
- 紧急模式触发后有效求助率。

### 10.2 功能指标
- 语音识别成功率（ASR success rate）。
- 首次响应时延（P95）。
- 提醒完成率（24h 内）。
- 应急事件平均响应时长（触发到首次动作）。

### 10.3 质量指标
- 7 日留存。
- 对话中断率。
- 用户主动差评率。
- 紧急模式误触发率/漏触发率。

### 10.4 建议埋点事件
- `voice_start`
- `voice_asr_success`
- `voice_asr_fail`
- `chat_response_ready`
- `emergency_detected`
- `emergency_call_120_click`
- `emergency_contact_call_click`
- `reminder_create`
- `reminder_done`

## 11. 非功能需求
- 性能：首屏可交互 <= 3s（4G 环境目标），关键接口 P95 <= 800ms。
- 可用性：月可用性目标 >= 99.5%（MVP 可放宽但需监控）。
- 兼容性：主流移动浏览器（微信内置、Safari、Chrome）。
- 可部署性：支持容器化部署，前后端分离，HTTPS 必选。

## 13. 验收标准（MVP）
1. 老人用户在 3 分钟内可完成首次使用与首次对话。
2. 高风险语句可在 1 次对话内触发紧急模式并给出明确动作。
3. 提醒创建到执行闭环可用，完成率可统计。
4. 对话与应急事件可追溯查询，字段完整率 > 98%。
5. 全链路具备基础日志与埋点，支持问题回放。

## 14. 待决策事项（需主代理协同确认）
- 家属端采用独立 H5 还是后台管理页面复用。

- 数据保留周期默认值（建议 180 天，可配置）。
