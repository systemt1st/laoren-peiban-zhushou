# 老人陪伴助手团队协作建议（产品与文档线）

## 1. 团队角色建议
- 产品经理（1）：负责需求收敛、PRD、验收标准、迭代节奏。
- 交互设计（1）：负责老人友好交互、页面原型、可用性测试。
- 前端工程师（1-2）：负责 H5 交互实现、语音能力接入、埋点。
- 后端工程师（1-2）：负责 API、数据模型、事件追溯、权限。
- 测试工程师（1）：负责功能、回归、兼容、风险场景验证。
- 运营/客服（1）：负责应急词库维护、事件回访与质检。

## 2. 里程碑职责划分
### M0（方案阶段）
- 产品：冻结 PRD 与验收标准。
- 设计：输出低保真到高保真原型。
- 技术：完成接口与数据库评审。

### M1（MVP 研发）
- 前端：完成首页/对话/提醒/应急/我的五大页面。
- 后端：完成会话、提醒、应急、联系人 API 与持久化。
- 测试：建立核心用例与冒烟脚本。

### M2（上线优化）
- 运营：上线应急词库与回访 SOP。
- 产品+数据：基于埋点调整流程与文案。

## 3. 协作机制
- 每日站会：15 分钟，聚焦阻塞项与风险项。
- 每周评审：需求变更、指标复盘、应急案例复盘。
- 交付门禁：功能完成必须同时满足“文档、测试、埋点”三项。

## 4. 风险协同清单
- 医疗合规风险：需在上线前完成法务文案审查。
- 应急链路风险：需预演“无法联网/无人接听”降级流程。
- 识别准确率风险：需要留出词库调优周期。

## 5. 发布前检查项
- 必须完成高危场景演练（胸痛、摔倒、呼吸困难）。
- 必须验证主流移动浏览器兼容性。
- 必须确认日志脱敏与数据保留策略。
